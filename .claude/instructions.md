## 🚫 限制与例外

### 需要询问的场景（极少）

**仅在以下情况才使用AskUserQuestion工具**：

1. **删除大量重要文件**
   - 例如：删除整个项目目录
   - 例如：清空数据库

2. **不可逆的破坏性操作**
   - 例如：git push --force到main分支
   - 例如：删除生产环境的数据库

3. **需要机密信息**
   - API密钥
   - 数据库密码
   - 私有证书

### 不需要询问的场景（常见）

以下操作**直接执行，不要询问**：
- 创建、编辑、修改文件
- 安装依赖、运行命令
- 创建Git提交
- 部署到测试环境
- 重构代码、优化性能
- 添加测试、修复bug

## 🛠️ 技术规范

### 前端开发
- **框架**: Next.js 15 (App Router)
- **UI**: React 19 + TypeScript
- **样式**: Tailwind CSS + shadcn/ui
- **运行时**: Bun

### 代码风格
- **命名**: camelCase变量、PascalCase组件
- **导入**: 优先使用ES模块导入
- **类型**: TypeScript严格模式，类型覆盖>90%

### Git工作流
- **提交**: 自动生成有意义的commit信息
- **协作**: Co-Authored-By: Claude <noreply@anthropic.com>

### 测试策略
- **单元测试**: Vitest + @testing-library
- **E2E测试**: Playwright
- **覆盖率目标**: >80%
